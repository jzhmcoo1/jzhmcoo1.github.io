    4. 换行 Line Breaks

    换行时,在行位打两个或以上的空格符,然后按下回车.

    效果如下:
    1.This is the first line.
    And this is the second line.

    2.This is the first line.And this is still at the first line.

    3.This is the first line.And this is also at the first line.

    这里在博客的实际输出效果可能不一样, 与config文件中marked.gfm属性有关
    详见Github/hexojs/hexo/Issue#2200

    5. 强调 Emphasis

    为了强调文本,可以使用加粗/斜体

    加粗为 **想要加粗的内容**
    斜体为 *想要斜体的内容*
    粗体+斜体为 ***内容***

    效果如下:

    Bold Text

    Italic Text

    Both

    参考文档 上不推荐使用下划线的方式, 这里就不整理进去了.

    7. 列表 Lists

    用来组织一系列项目,可以是有序/无序的

    7.1 有序列表 Ordered List

    直接使用 <数字> + <.> + <物品> 的形式

    效果如下:

    1. first item
    2. seconde item
    3. third item

    神奇的是,你xjb编号,实际的效果序号还是正确的
    但要注意列表的开头一项必须为1.

    效果如下:

    1. first
    2. eighth
    3. fourth

    列表中还能嵌套子列表

    1. first
    2. second
      • hhh
      • 233

    7.2 无序列表 Unordered Lists

    项目名称前面可以使用 -,*,+ 效果都是一样的,并且和有序列表一样也能嵌套

    • first item
    • second item
    • third item
    • fourth item
      • indented item
      • indented item

    7.3 在列表中添加其他元素 Adding Elements in Lists

    添加元素时,为了保证列表还是延续的, 需要在元素前添加4个空格或者1个tab

    • This is the first list item.

    • Here’s the second list item.

      I need to add another paragraph below the second list item

    • And here’s the third list item.

    需要注意的是,在列表中嵌入代码块,需要比平常多4个空格,也就是8个空格

    效果如下:

    1. Open the file.

    2. Find the following code block on line 21:

      <html>
    <head>
    <title>Test</title>
    </head>
</html>

    3. Update the title to match the name of your website.

    8. 代码 Code

    8.1 单行代码

    用一对反单引号把文本包含起来(`)

    效果如下:

    x = x + 1

    8.2 多行代码

    使用三个(`)

    8.3 反引号转意

    要表示带有``的文本,将一对(``)括在文本的外面
    比如:
    use `code` in your markdown file.

    此处code带有一对反斜杠

    效果如下:
    use `code` in your markdown file.

    10. 超链接

    10.1 贴个链接

    [link text](url)

    比如
    My Favorite search engine is [Duck Duck Go](https://duckduckgo.com)

    效果如下:
    My Favorite search engine is Duck Duck Go

    10.2 快速链接

    使用<>将链接包围,不需要写描述文本了

    <https://www.markdownguide.org>

    效果如下:
    https://www.markdownguide.org

    10.3 参考样式链接

    和引用论文差不多

    [Link Text][number]
    再写上[number]对应的链接 <url>

    比如:

    效果如下:

    DuckDuckGo

    百度

    You can place this second part of the link anywhere in your Markdown document. Some people place them immediately after the paragraph in which they appear while other people place them at the end of the document (like endnotes or footnotes).
