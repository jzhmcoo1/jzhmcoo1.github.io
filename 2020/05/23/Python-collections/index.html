Python collections summary

前言

最近在刷力扣题，我一般会使用 python3进行解答，但是看题解的时候，看到 collections 这个内置库经常经常被调用，想必它的实用性和重要性就非常之高了，今天我就根据官方文档来总结一下 collections 库里常用的类和方法。

collections ————容器数据类型

Source code: Lib/collections/init.py

这个模块实现了特定目标的容器，以提供 python 标准内建容器dict,list,set和tuple的替代选择。

Counter 对象

一个计数器工具提供快速和方便的计数

跟着文档的例子敲一遍

这里实现的是对列表中出现的元素进行计数

class collections.Counter([iterable-or-mapping])

一个Counter是一个dict的子类，用于计数可哈希对象。它是一个集合，元素像字典键一样存储，它们的计数存储为值。计数可以是任何整数值，包括 0 和负数。Counter类有点像其他语言中的 bags或multisets

Counter对象有一个字典接口，如果引用的键没有任何记录，就返回一个0，而不是弹出一个KeyError

设置一个计数为0不会从计数器中移去一个元素。使用 del 来删除它:

3.7版更改： 作为 dict 的子类，Counter 继承了记住插入顺序的功能。 Counter 对象进行数学运算时同样会保持顺序。 结果会先按每个元素在运算符左边的出现时间排序，然后再按其在运算符右边的出现时间排序。

计数器除了字典方法外，还提供了三个其他方法：

  • elements()

    返回一个迭代器，其中每个元素将重复出现计数值所指定次。元素会按首次出现的顺序返回，如果一个元素的计数值小于 1，elements()将会忽略它。

  • most_common([n])

    返回一个列表，其中包含 n 个最常见的元素及出现次数，按常见程度由高到底排序。如果 n 被省略或为None，most_common()将回复计数器中的 所有元素。计数值相等的元素按首次出现的顺序排序：

  • subtract([iterable-or-mapping])

    从迭代对象或映射对象减去元素。像 dict.update()但是是减去，而不是替换。

    输入和输出都可以是 0 或负数。

OrderedDict

5.25 日力扣的签到题，设计了一个 LRU 算法，虽然上过操作系统，知道是怎么回事，但具体实现起来并不清楚用什么数据结构。想要做到$O(1)$的增加 key 和查找 key 的操作，官方题解给出的是双向链表+哈希表的做法，正好 python 中就有这种封装的数据结构。

class collections.OrderedDict([items])

  • popitem(last=True)

    这个方法将会移除字典中的一对键值对，如果 last=True则是后进先出（移除最后一组进入的键值对），否则是先进先出（移除最先进来的键值对）

  • move_to_end(key,last=True)

    将已存在的 key 移动至有序字典的任意一端，默认 last=True，将会移动到最右边一端，而 last=false 则移动到最左边一端，如果键不存在，则会报出KeyError的错误
